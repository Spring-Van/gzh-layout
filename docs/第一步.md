很好，既然你已经把 **Electron + Vue + Vite** 搭起来了，那下一步就是把“选择文件夹后的一整套流程”串起来。

我会按 **最实战的方式** 给你讲：

1. 先实现最小闭环
2. Electron 主进程怎么选文件夹
3. preload 怎么暴露 API
4. Vue 页面怎么调用
5. 选完文件夹后怎么扫描图片
6. 怎么形成“项目”
7. 后续步骤怎么串成流水线

# 一、你现在要做的最小闭环

先不要一上来就做“备份 + 去重 + 排版 + 导出”全部。

建议先实现这条最短路径：

> 点击“选择文件夹” → Electron 打开目录选择器 → 返回路径 → 扫描目录里的图片 → 返回图片列表 → 前端展示

只要这个跑通，你后面的备份、去重、排版都能逐步加。

---

# 二、完整流程应该怎么设计

我建议你把“选择文件夹后”的流程设计成 5 步：

## Step 1：选择文件夹

前端点击按钮，调用 Electron 主进程打开系统目录选择器。

## Step 2：创建项目

拿到文件夹路径后，生成一个项目对象：

- projectId
- 项目名
- folderPath
- 创建时间

## Step 3：扫描图片

主进程遍历文件夹中的图片，提取：

- 文件名
- 路径
- 文件大小
- 宽高
- 格式

## Step 4：保存项目数据

把项目和图片列表先存到前端状态里，后面再落 SQLite。

## Step 5：跳转到项目工作台

页面显示：

- 左侧图片列表
- 中间预览区
- 右侧配置区

## 1. 主进程里注册 IPC

假设你的主进程在：

```bash
electron/main/index.ts
```

你加一个 IPC：

```ts
import { app, BrowserWindow, dialog, ipcMain } from "electron";
import path from "path";

let mainWindow: BrowserWindow | null = null;

function createWindow() {
  mainWindow = new BrowserWindow({
    width: 1400,
    height: 900,
    webPreferences: {
      preload: path.join(__dirname, "../preload/index.js"),
    },
  });

  mainWindow.loadURL("http://localhost:5173");
}

app.whenReady().then(() => {
  createWindow();

  ipcMain.handle("file:selectFolder", async () => {
    const result = await dialog.showOpenDialog({
      properties: ["openDirectory"],
    });

    if (result.canceled || result.filePaths.length === 0) {
      return null;
    }

    return result.filePaths[0];
  });
});
```

## 3. 主进程写扫描函数

建议单独抽 service，先简单写在主进程也行。

```ts
import fs from "fs-extra";
import path from "path";
import sharp from "sharp";
import { nanoid } from "nanoid";

const IMAGE_EXTENSIONS = [".jpg", ".jpeg", ".png", ".webp"];

async function scanImagesInFolder(folderPath: string) {
  const files = await fs.readdir(folderPath);

  const imageList = [];

  for (const fileName of files) {
    const fullPath = path.join(folderPath, fileName);
    const stat = await fs.stat(fullPath);

    if (!stat.isFile()) continue;

    const ext = path.extname(fileName).toLowerCase();
    if (!IMAGE_EXTENSIONS.includes(ext)) continue;

    try {
      const metadata = await sharp(fullPath).metadata();

      imageList.push({
        id: nanoid(),
        name: fileName,
        path: fullPath,
        size: stat.size,
        width: metadata.width || 0,
        height: metadata.height || 0,
        format: metadata.format || ext.replace(".", ""),
      });
    } catch (error) {
      console.error("图片解析失败:", fullPath, error);
    }
  }

  return imageList;
}
```

## 7. 前端调用扫描

```vue
<script setup lang="ts">
import { ref } from "vue";

interface ImageItem {
  id: string;
  name: string;
  path: string;
  size: number;
  width: number;
  height: number;
  format: string;
}

const folderPath = ref("");
const images = ref<ImageItem[]>([]);
const loading = ref(false);

const handleSelectFolder = async () => {
  const selected = await window.electronAPI.selectFolder();
  if (!selected) return;

  folderPath.value = selected;
  loading.value = true;

    const result = await window.electronAPI.scanFolder(selected);
    images.value = result;

    loading.value = false;
  }
};

    <p v-if="folderPath">当前文件夹：{{ folderPath }}</p>
    <p v-if="loading">扫描中...</p>

    <ul>
      <li v-for="item in images" :key="item.id">
        {{ item.name }} / {{ item.width }}x{{ item.height }} / {{ item.format }}
      </li>
    </ul>
  </div>
</template>
```

# 十、建议你现在就抽一个 projectService

前端不要把这些全堆在页面里。  
建议你在 `src/api/native.ts` 或 `src/services/project.ts` 里封装流程。

例如：

```ts
export async function createProjectFromFolder() {
  const folderPath = await window.electronAPI.selectFolder();
  if (!folderPath) return null;

  const images = await window.electronAPI.scanFolder(folderPath);

  const project = {
    id: crypto.randomUUID(),
    name: folderPath.split(/[/\\]/).pop() || "未命名项目",
    sourceFolder: folderPath,
    createdAt: new Date().toISOString(),
  };

  return {
    project,
    images,
  };
}
```

页面只负责：

```ts
const handleCreateProject = async () => {
  const result = await createProjectFromFolder();
  if (!result) return;

  projectStore.setCurrentProject(result.project);
  imageStore.setImages(result.images);
  router.push(`/project/${result.project.id}`);
};
```

# 十一、后续每一步应该怎么加

当“选文件夹 → 扫描 → 展示”完成后，按这个顺序继续：

---

## 第一步：自动备份

新增 API：

- `backupProjectFolder(sourceFolder, projectId)`

主进程实现：

- 创建应用数据目录
- 复制原始文件夹进去

---

## 第二步：生成缩略图

新增 API：

- `generateThumbnail(imagePath)`

或者扫描时顺便生成缓存缩略图。

这样图片列表更快。

---

## 第三步：去重

新增 API：

- `findDuplicateImages(projectId)`

先做最简单版本：

- MD5 去重

---

## 第四步：排序和封面

前端本地处理即可：

- 手动拖拽排序
- 设置 `isCover = true`

---

## 第五步：模板预览

前端根据：

- title
- summary
- coverImage
- images

生成 HTML 字符串，用 iframe 预览。

## 第六步：导出 HTML

主进程写文件到 export 目录。
