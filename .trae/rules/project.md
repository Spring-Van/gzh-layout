# 项目规则

## 核心原则（始终生效）

- ‌优先使用 TypeScript（严格模式）‌
- ‌优先函数式 + hooks 写法，class 组件仅在极特殊场景使用‌
- ‌组件尽量小而纯（单一职责），单个文件不超过 300–400 行‌
- UI 优先使用可组合、可复用的原子组件 + 组合组件模式
- ‌状态管理 Pinia
- ‌样式：优先 Tailwind CSS
- ‌代码风格严格遵循 ESLint + Prettier + typescript-eslint 推荐规则‌
- ‌永远写 JSDoc / TSDoc 注释（尤其是工具函数、hooks、复杂组件）‌
- ‌优先使用现代语法（可选链、nullish 合并、top-level await 等）
